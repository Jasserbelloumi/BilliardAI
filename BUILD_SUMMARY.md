# ملخص أدوات وتقنيات البناء لمشروع BilliardAI

بناءً على الخطوات التي قمنا بها معاً في Termux، إليك قائمة دقيقة بالأدوات والتقنيات التي استخدمناها لبناء التطبيق:

1. **بيئة التطوير المحلية (على الهاتف)**
   - **Termux:** المحاكي الذي استخدمناه كـ Terminal لإدارة الملفات وكتابة الأوامر.
   - **Git:** تتبّع التغييرات في الكود وحفظ النسخ.
   - **GitHub CLI (gh):** التحكم بحساب GitHub، إنشاء المستودعات، وتحميل ملفات الـ APK.

2. **لغة البرمجة والهيكلية**
   - **Java:** اللغة الأساسية لكود التطبيق (`MainActivity.java`).
   - **XML:** تعريف الـ Manifest والموارد.
   - **WebView:** محرك لعرض محتوى ويب داخل التطبيق (إذا استُخدم).

3. **أدوات بناء المشروع**
   - **Gradle:** نظام تجميع الكود وتحويله إلى APK.
   - **JDK 17:** حزمة تطوير جافا اللازمة لعمل Gradle.

4. **نظام البناء السحابي**
   - **GitHub Actions:** السيرفر الخارجي الذي يقوم بعملية البناء بدلًا من الجهاز المحلي.
   - **YAML (Workflow):** وصف خطوات البناء (تثبيت JDK، SDK، تشغيل ./gradlew assembleDebug، رفع الـ Artifact).

**ملخّص العملية في سطر واحد:**  
نكتب الكود في Termux → نرفعه إلى GitHub → GitHub Actions يقوم بالبناء باستخدام Gradle → نحمل الـ APK النهائي بواسطة `gh`.
